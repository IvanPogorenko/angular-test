1. Как вы можете описать процесс распространения событий в
JavaScript (Event Propagation)? Расскажите о его основных фазах.
Какие есть способы применения знаний о фазах
рааспространения событий на практике?

Ответ:
Процесс распространения событий в JS состоит из трех фаз: от корня документа к целевому элементы, достижение целевого элемента
и движение от целевого вверх к корню документа
практическое применение: делегирование событий, прекращение распространение (stopPropagation()), предотвращение поведения
по умолчанию (preventDefault())

2. Расскажите, что такое Promise в JavaScript, как он работает, какие
основные состояния у него бывают? Каким еще способом можно
обработать асинхронный код? Какую роль в асинхронности играет
событийный цикл (Event Loop)?

Ответ:
Промис - это объект, предоставляющий результат асинхронной операции, которая может завершиться с ошибкой (reject) или успешно (resolve).
Три основных состояния: начальное (операция еще не завершена), операция успешна (вызов resolved), операция упала с ошибкой (вызов reject)
три основных метода: .then() (при успехе), .catch() (при ошибке), .finally() (при любом завершении)
Также можно формировать цепочки промисов: promise.then().then().then.catch()
Также асинхронный код можно обработать callback (старый метод, трудночитаемый) и async/await (предпочтительный)
Event Loop — это механизм, который позволяет JavaScript выполнять асинхронный код в однопоточной модели
Event Loop: не блокирует основной поток, управляет очередями задач и позволяет работать с сетью без зависаний
(грубо говоря можем выполнять действия на сайте пока грузятся данные)

3. Что такое ООП, и какие его ключевые принципы вы могли бы
описать? При объяснении приводите примеры. Максимально
подробно расскажите о том, как ООП реализовано в JavaScript.

ООП - подход к программированию, в котором код строится как набор классов, которые взаимодействуют друг с другом
4 принципа:
1) инкапсуляция - объединение атрибутов и методов для обработки в одном объекте и скрытие деталей работы
2) наследование - создание новых классов на основе уже существующих с переопределением методов и\или полей
3) полиморфизм - объекты разных классов могут реагировать на один и тот же вызов метода по-разному
4) абстракция - выделение главных характеристик объекта и скрытие деталей реализации
ООП в JS:
прототипное наследование через prototype, а классы - это синтаксический сахар над прототипами
приватные поля и метода через символ #
также есть constructor
обращение к полям и методам текущего класса через this
использование Миксинов Object.assign()
использование композиции вместо наследования

4. Опишите максимально детально, как браузер обрабатывает URL,
начиная с момента его ввода в адресную строку и заканчивая
отображением страницы. Какие технологии позволяют ускорить
процесс? Какие проблемы (безопастность, междоменное
взаимодействие) могут возникать в процессе?

Ответ:
1. браузер разбивает url на компоненты: протокол (https) и домен (www.qwe.com)
2. поиск ip адреса соответствующего домену. Поиск в кэше, если не нашел, то запрос на ip уходит к DNS серверам
3. получив ip устанавливается соединение с сервером (TCP протокол)
4. определение портов для определения протокола передачи данных (HTTP/HTTPS)
5. настройка безопасности SSL (все данные между клиентом и сервером передаются в зашифрованном виде)
6. браузер отправляет на сервер http запрос для получения контента
7. сервер отдает html, css, изображения, js код
8. рендер страницы
Ускорение:
1) использование CDN - ускоряет доставку статики
1) кэширование ресурсов
1) предзагрузка ресурсов
Проблемы безопасности:
CORS - браузер блокирует междоменные запросы (решается на сервере)
XSS атаки (решается экранированием данных)
